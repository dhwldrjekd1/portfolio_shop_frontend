│   ├── AppHeader.vue         # 헤더 (로그인 모달, 햄버거 메뉴 포함)

    └── index.js              # 라우터 설정 (가드 포함)

## 구현 기능

### 회원
- 회원가입 / 로그인 / 로그아웃 — 로그아웃 시 서버 세션도 함께 무효화(`/api/member/logout` 호출)
- 마이페이지 (회원정보 조회/수정, 회원탈퇴)
- 비밀번호 정책 사전 검증 (8자 이상 + 영문 소문자·숫자·특수문자, 회원가입/마이페이지 비밀번호 변경 모두 적용) — 서버 정책과 동일 기준으로 즉시 안내
- 비밀번호 찾기 (임시 비밀번호를 이메일로 발송, 화면/응답에는 노출하지 않음)
- 구매금액 기반 자동 등급 업데이트 (브론즈/실버/골드/플래티넘)
- 라우터 가드 (비로그인/비관리자 접근 차단) — 화면 접근 차단은 프론트, 실제 데이터 접근 권한은 백엔드 세션 기반 검증(`SessionAuth`)이 이중으로 보장

### 상품
- 상품 목록 (카테고리/뱃지/가격/평점 필터, 정렬)
- 상품 상세 (이미지 갤러리, 색상/사이즈 선택)
- 할인가 표시 (원가 취소선 + 할인가 + %OFF)
- DB 기반 재고/할인율/가격/별점 실시간 반영

### 장바구니 / 위시리스트 / 결제
- 장바구니 담기 / 수량 변경 / 삭제 (DB 기반, 새로고침 후에도 유지)
- 장바구니 합계에 할인율 반영(100원 단위 반올림, 상품 카드/상세와 동일 규칙) — 결제 금액도 서버가 동일 기준으로 재계산해 대조
- 위시리스트 담기 / 제거 (DB 기반, 새로고침 후에도 유지)
- 3단계 결제 프로세스 (배송정보 → 결제수단 → 확인)
- 토스페이먼츠 결제 연동
- 주문 취소 (주문접수/배송중 상태만 가능)

### 보안
- CSRF 토큰 자동 첨부 — `csrf.js`가 `fetch`를 전역으로 감싸 상태변경 요청(POST/PUT/DELETE)마다 서버가 발급한 `XSRF-TOKEN` 쿠키 값을 `X-XSRF-TOKEN` 헤더로 첨부

### 리뷰
- 배송완료 주문 확인 후 리뷰 작성
- 동일 상품 중복 리뷰 방지
- 리뷰 수정 / 삭제
- DB 기반 평균 별점 표시

### 고객지원 / 커뮤니티
- 공지사항 / QnA / 고객문의 / 커뮤니티 게시판

### 관리자 대시보드 (6개 탭)
- 회원 목록 / 주문 관리 / 재고 관리
- 판매 관리 / 상품 관리 / 리뷰 관리

### 반응형 (모바일)
- 전체 페이지 모바일 대응 (max-width: 768px)
- 햄버거 메뉴 (모바일 네비게이션)
- 관리자 테이블 핵심 컬럼만 표시
- iOS Safari 자동 줌인 방지

## 트러블슈팅

- **장바구니/결제 금액 불일치** (2026-07-19) — 할인 상품을 담으면 개별 상품가(원가 기준)와 하단 합계(할인 반영 기준)의 계산 방식이 달라 화면에 표시되는 금액이 서로 어긋나던 문제. `getDiscountedPrice()` 공용 함수를 추가해 장바구니/결제/주문완료 화면의 가격 계산을 하나로 통일.
- **결제 성공 후 주문 저장 실패 시 무응답** (2026-07-19) — 토스페이먼츠 결제 승인은 됐는데 이어지는 주문 저장 요청이 실패해도(예: 재고 부족) 아무 안내 없이 "결제 완료" 화면 문구만 표시되던 문제. 저장 실패 시 에러 안내와 함께 마이페이지로 이동시키고, 장바구니는 비우지 않아 재주문이 가능하도록 하며, 남아있던 임시 주문 데이터(세션)도 함께 정리하도록 수정.
- **토스 결제 시 결제 승인 번호가 주문 요청에 누락** (2026-07-19) — 백엔드가 결제 승인 기록과 주문을 대조하도록 바뀌면서, 결제 성공 후 주문 생성 요청에 토스 주문번호(`tossOrderId`)를 함께 보내도록 수정. (관련 백엔드 변경: 결제 승인 기록 저장 및 주문 생성 시 검증)
- **할인가 계산 로직 중복** (2026-07-19) — `ProductCard.vue`, `ProductDetailView.vue`가 장바구니/결제 화면과 별개로 할인가 계산식을 각자 구현하고 있던 것을 `store.getDiscountedPrice()` 공용 함수를 쓰도록 통일. 계산 결과 자체는 기존과 동일.
- **관리자 상품 저장에 확인창 없음** (2026-07-19) — `AdminView.vue`의 상품 재고/할인율 저장 버튼만 다른 관리자 액션(주문삭제, 리뷰삭제, 권한변경 등)과 달리 확인창 없이 바로 반영되던 문제. 동일한 스타일로 `confirm()` 추가.
- **위시리스트 할인가 미표시** (2026-07-20) — 위시리스트 목록이 스토어의 원본 상품 데이터를 그대로 써서, 존재하지 않는 필드(`item.originalPrice`)와 비교하는 조건 때문에 할인 취소선과 할인가가 전혀 표시되지 않던 문제. `getDiscountedPrice()` 공용 함수로 통일해 상품카드/상세/장바구니/결제와 동일하게 표시.
- **결제 승인 후 주문 생성 단계에서 예외 발생 시 처리 누락** (2026-07-20) — 주문 생성 요청이 실패 응답(`success:false`)이 아니라 네트워크 오류 등 예외로 끊기는 경우, 카드 결제는 이미 완료됐는데도 "결제 승인 실패" 안내와 함께 결제 화면으로 돌려보내던 문제. 주문 생성 단계를 별도로 감싸 결제 완료 이후의 실패는 항상 동일하게(안내 + 세션 정리 + 마이페이지 이동) 처리하도록 수정.
- **관리자 등급/주문상태 변경에 확인창 없음** (2026-07-20) — 회원 등급 변경, 주문 상태 변경이 다른 관리자 액션과 달리 확인창 없이 select 변경 즉시 반영되던 문제. 동일한 스타일로 `confirm()` 추가.
